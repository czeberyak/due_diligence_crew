# 🏭 Due Diligence Crew

**Multi-agent система на базе CrewAI и OpenRouter для автоматизированного Due Diligence инвестиционных ТЭО**

Проект реализует концепцию **экспертного консилиума** под управлением LLM-менеджера, который автоматизирует глубокий технический, экологический и финансовый аудит проектной документации крупных промышленных объектов.

---

## 🎯 Ключевые возможности

- **🔍 Глубина аудита:** Полный комплаенс-контроль проектной документации на соответствие ФНП Ростехнадзора, ГОСТ, ТУ и ISO.
- **⚖️ Физическая верификация:** Автоматический сквозной расчет материально-тепловых балансов с жестким отсевом невязок (порог > 5%).
- **💰 Экономический перевод:** Маппинг инженерных уязвимостей в стоимостную оценку рисков модернизации, штрафов и изменения CAPEX/OPEX.
- **🌐 Архитектурная отказоустойчивость:** Полная независимость от прямых зарубежных блокировок OpenAI за счет интеграции шлюза OpenRouter.
- **📊 Иерархическое управление:** LLM-менеджер самостоятельно декомпозирует задачу, распределяет подзадачи между экспертами и проводит кросс-валидацию.

## 🏗️ Архитектура консилиума

Система построена по канонам модульных production-систем с разделением фабрик агентов, логики задач и расчетных инструментов:

```
due_diligence_crew/
├── data/
│   ├── 01_raw/              # Исходные PDF-файлы ТЭО
│   └── 02_processed/        # Сгенерированные отчеты (Markdown)
├── src/
│   ├── agents.py            # Фабрика агентов (Auditor, Engineer, Analyst)
│   ├── tasks.py             # Конфигурация конвейера задач и иерархических связей
│   ├── tools.py             # Кастомные инструменты (PDF-парсер, Мат. валидатор)
│   └── main.py              # Точка входа, конфигурация LLM и запуск Crew
├── .env                     # Переменные окружения и API-ключи
├── check_imports.py         # Модуль предполётной валидации
└── requirements.txt         # Зависимости (CrewAI, Pydantic v2, pdfplumber)
```

### 👥 Роли агентов

| Агент | Роль | Инструменты |
|-------|------|-------------|
| **🛡️ Safety Auditor** | Главный эксперт по ПБ и экологическому комплаенсу | `DocumentSearchTool` |

## 🛠️ Кастомные инструменты

### `DocumentSearchTool`
Честный парсер проектной документации на базе `pdfplumber`. Постранично извлекает текст и **технологические таблицы** (критично для материальных балансов!) из PDF, фильтруя данные по ключевым словам. Поддерживает защитные механизмы от системных файлов Windows и лимит на объем токенов.

### `ProcessCalculatorTool`
Жестко детерминированный верификатор на чистом Python. Агент-технолог **обязан** прогонять числовые данные схем через этот инструмент. Автоматически вычисляет:
- Суммарные потоки сырья и продуктов
- Процент невязки материального баланса
- Фактический vs заявленный выход целевого продукта
- **Вердикт:** при невязке > 5% — автоматический флаг физической нереализуемости проекта

## 🏆 Результаты успешного прогона (Case Study)

Система успешно провела Due Diligence ТЭО проекта строительства завода по производству полиэтилена низкого давления (500 000 т/год).

### Ключевые findings консилиума:

| Показатель | Заявлено в ТЭО | Выявлено консилиумом |
|------------|----------------|----------------------|
| **Выход целевого продукта** | 95% | **91.24%** (отклонение -3.76%) |
| **Невязка материального баланса** | — | **2.37%** (в пределах нормы ✅) |
| **Базовый CAPEX** | — | $1 200 млн |
| **Скорректированный CAPEX** | — | **$1 269 млн** (+$69 млн на mitigation) |
| **Базовый OPEX** | — | $420 млн/год |
| **Скорректированный OPEX** | — | **$630 млн/год** (+$210 млн резерв) |
| **IRR (базовый)** | 18.5% | **18%** |
| **IRR (с учётом рисков)** | — | **≈ 12%** |
| **Срок окупаемости** | 6.2 года | **9.5 года** |

### Выявленные критические риски:
1. 🔴 **Нестабильность полимеризации** (вероятность 15%) — требует CAPEX +$20 млн на онлайн-мониторинг
2. 🔴 **Сверхнормативные выбросы** (CO, VOC, NOx) — CAPEX +$10 млн на скрубберы
3. 🔴 **Недостаточная защита АСУ ТП** (ГОСТ Р 53684-2009) — CAPEX +$9 млн на кибер-защиту
4. 🟡 **Отказ оборудования** — CAPEX +$15 млн на резервные компрессоры
5. 🟡 **Рост цен на сырьё** (10-15%) — OPEX +$45 млн/год

## 🚀 Быстрый старт

### 1. Клонирование и установка зависимостей

```bash
git clone https://github.com/czeberyak/due_diligence_crew.git
cd due_diligence_crew
python3 -m venv .venv
source .venv/bin/activate  # для Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Конфигурация окружения

Создайте файл `.env` в корне проекта:

```env
OPENROUTER_API_KEY=sk-or-v1-ваш-ключ-из-панели-openrouter
```

### 3. Предполётная проверка

```bash
python check_imports.py
```

Ожидаемый результат: `✅ Все базовые компоненты архитектуры связаны корректно!`

### 4. Запуск консилиума

Поместите ваш PDF-файл с ТЭО в `data/01_raw/` и выполните:

```bash
python src/main.py
```
