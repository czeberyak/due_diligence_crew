# 🏭 Due Diligence Crew

**Multi-agent система на базе CrewAI и OpenRouter для автоматизированного Due Diligence инвестиционных ТЭО**

Проект реализует концепцию **экспертного консилиума** под управлением LLM-менеджера, который автоматизирует глубокий технический, экологический и финансовый аудит проектной документации крупных промышленных и энергетических объектов.

---

##  Ключевые возможности

- **🔍 Глубина аудита:** Полный комплаенс-контроль проектной документации на соответствие ФНП Ростехнадзора, ГОСТ, ТУ, ISO и отраслевым регламентам.
- **️ Физическая верификация:** Автоматический сквозной расчет материальных и энергетических балансов с жестким отсевом невязок и детектированием нарушения законов физики (выход >100%, КИУМ >25% для СЭС).
- **📜 Multi-Document RAG с категорийным поиском:** Автоматический кросс-чекинг ТЭО с реальной базой нормативных актов, структурированной по 8 отраслевым категориям.
- **💰 Экономический перевод:** Маппинг инженерных уязвимостей в стоимостную оценку рисков модернизации, штрафов и изменения CAPEX/OPEX в рублях.
- **🌐 Архитектурная отказоустойчивость:** Полная независимость от прямых зарубежных блокировок OpenAI за счет интеграции шлюза OpenRouter.
- **📊 Иерархическое управление:** LLM-менеджер самостоятельно декомпозирует задачу, распределяет подзадачи между экспертами и проводит кросс-валидацию.
- **🚀 Пакетный режим:** Поддержка массового анализа нескольких ТЭО одновременно с автоматическим сохранением отчетов.

## ️ Кастомные инструменты

### `DocumentSearchTool`
Честный парсер проектной документации на базе `pdfplumber`. Постранично извлекает текст и **технологические таблицы** (критично для материальных балансов!) из PDF, фильтруя данные по ключевым словам. Поддерживает защитные механизмы от системных файлов Windows и лимит на объем токенов (15 000 символов).

### `RegulatorySearchTool`
Инструмент Multi-Document RAG для сверки с законодательством. Сканирует PDF-файлы в **выбранной категории** нормативной базы (или во всех сразу), ищет по ключевым словам в реальных текстах нормативных актов. Позволяет агенту цитировать конкретные требования законов вместо галлюцинаций.

**Поддерживаемые категории:**
- `01_construction` — Градостроительство, землеотвод, разрешения на строительство
- `02_industrial_safety` — Промышленная безопасность ОПО (химия, нефтегаз)
- `03_ecology` — Экология, ПДВ, отходы (универсальная)
- `04_pressure_equipment` — Оборудование под давлением
- `05_automation` — АСУ ТП, кибербезопасность
- `06_labor` — Охрана труда (универсальная)
- `07_fire_safety` — Пожарная безопасность (универсальная)
- `08_energy` — Энергетика, ПУЭ, подключение к сетям (для СЭС, ВЭС, ТЭЦ)

### `ProcessCalculatorTool`
Жестко детерминированный верификатор на чистом Python для **химических и производственных проектов**. Агент-технолог **обязан** прогонять числовые данные схем через этот инструмент. Автоматически вычисляет:
- Суммарные потоки сырья и продуктов
- Процент невязки материального баланса
- Фактический vs заявленный выход целевого продукта
- **Вердикт:** при невязке > 5% — автоматический флаг физической нереализуемости проекта
- **Защита от «создания материи»:** если масса продуктов > массы сырья — критическая ошибка физики

### `EnergyBalanceCalculatorTool`
Жестко детерминированный верификатор для **энергетических проектов** (СЭС, ВЭС, ТЭЦ). Проверяет:
- Реальную выработку на основе КИУМ (коэффициент использования установленной мощности)
- Соответствие заявленной выработки расчетной
- **Вердикт:** при КИУМ > 25% для солнца или > 35% для ветра — автоматический флаг физической нереализуемости

## 🏆 Результаты успешных прогонов (Case Study)

Система провела серию итеративных Due Diligence на двух разных тематиках. Ниже — эволюция качества аудита.

### Итерация 1-3: Химическая промышленность (Завод полиэтилена 500 000 т/год)

| Показатель | Итерация 1 (базовый) | Итерация 2 (RAG) | Итерация 3 (физ. верификация) |
|------------|---------------------|------------------|-------------------------------|
| **Выход продукта** | 91.24% (отклонение -3.76%) | 91.24% | **104.2%** (масса продуктов > массы сырья) |
| **Невязка баланса** | 2.37% ✅ | 5.96% ⚠️ | ❌ **КРИТИЧЕСКАЯ ОШИБКА ФИЗИКИ** |
| **Нормативный анализ** | Галлюцинации (банковские ГОСТы) | 7 критических пунктов (ПСУ ПБ, ПМ ОПБ, ПТОТР) | ПСУ ПБ, ПМ ОПБ, ПТОТР, АОР/HAZOP, ПЧС, САО, ПМР, ПОО |
| **Доп. CAPEX** | $69 млн (галлюцинация USD) | +445 млн ₽ | 100% риск потери инвестиций |
| **Рекомендация** | Одобрить (слабо обосновано) | Приостановить финансирование | **Отклонить** (физически нереализуем) |

**Ключевые findings:**
- 🔴 Нестабильность полимеризации (вероятность 15%) — требует CAPEX +20 млн ₽ на онлайн-мониторинг
- 🔴 Сверхнормативные выбросы (CO, VOC, NOx) — CAPEX +10 млн ₽ на скрубберы
- 🔴 Недостаточная защита АСУ ТП (ГОСТ Р 53684-2009) — CAPEX +9 млн ₽ на кибер-защиту
- 🟡 Отказ оборудования — CAPEX +15 млн ₽ на резервные компрессоры
- 🟡 Рост цен на сырьё (10-15%) — OPEX +45 млн ₽/год

### Итерация 4: Энергетика (Солнечная электростанция 50 МВт) 

| Показатель | Заявлено в ТЭО | Выявлено консилиумом |
|------------|----------------|----------------------|
| **КИУМ** | 100% | **❌ КРИТИЧЕСКАЯ ОШИБКА ФИЗИКИ: максимум 25% для СЭС** |
| **Годовая выработка** | 438 000 МВт·ч | **109 500 МВт·ч** (в 4 раза меньше!) |
| **Базовый CAPEX** | 4.5 млрд ₽ | 4.5 млрд ₽ |
| **Скорректированный CAPEX** | — | **4.79 млрд ₽** (+292 млн ₽ на недостающие системы) |
| **NPV (20 лет, 8%)** | +28.5 млрд ₽ (галлюцинация) | **-3.84 млрд ₽** (отрицательный!) |
| **IRR** | 18.5% | **-2.3%** (отрицательный!) |
| **Срок окупаемости** | 6.2 года | **>30 лет** (не достигается) |

**Ключевые findings:**
- 🔴 **Физическая нереализуемость:** КИУМ=100% невозможен для солнечной генерации
- 🔴 **Отрицательный NPV:** Проект убыточен даже при консервативных оценках
- 🟡 **Недостающие системы:** ТТ/ТУ, грозозащита, пожарные системы, план утилизации панелей (+292 млн ₽ CAPEX)
- 🟡 **Нормативные риски:** Отсутствие ТТ/ТУ, задержка согласования с сетью (+176 млн ₽/год OPEX-риски)

**Рекомендация:** ❌ **Отклонить проект в текущем виде.** Требуется пересчет с реалистичным КИУМ ≤25%, привлечение гос. субсидий (ДПМ ВИЭ), оптимизация CAPEX.

## 🚀 Быстрый старт

### 1. Клонирование и установка зависимостей

```bash
git clone https://github.com/czeberyak/due_diligence_crew.git
cd due_diligence_crew
python3 -m venv .venv
source .venv/bin/activate  # для Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Конфигурация окружения

Создайте файл `.env` в корне проекта:

```env
OPENROUTER_API_KEY=sk-or-v1-ваш-ключ-из-панели-openrouter
CREWAI_TRACING_ENABLED=false
```

### 3. Предполётная проверка

```bash
python check_imports.py
```

Ожидаемый результат: `✅ Все базовые компоненты архитектуры связаны корректно!`

### 4. (Опционально) Индексация документов в ChromaDB

```bash
python src/ingest.py
```

### 5. Запуск консилиума

**Режим 1: Анализ конкретного файла**
```bash
python src/main.py --file data/01_raw/TEO/TEO_project_example.pdf
```

**Режим 2: Пакетный анализ всех ТЭО в папке**
```bash
python src/main.py --batch
```

**Режим 3: По умолчанию (анализ TEO_project_example.pdf)**
```bash
python src/main.py
```

Отчеты автоматически сохраняются в `data/02_processed/` с уникальными именами и датами.

## 📚 Управление нормативной базой

### Структура документов

Нормативные документы организованы по 8 категориям в `data/01_raw/normative_base/`:

| Категория | Назначение | Ключевые документы |
|-----------|------------|-------------------|
| `01_construction` | Градостроительство | Градостроительный кодекс РФ, СП 42.13330.2016 |
| `02_industrial_safety` | Промышленная безопасность | ФЗ-116, Приказ Ростехнадзора №533 |
| `03_ecology` | Экология | ФЗ-7, ФЗ-89, ГОСТ Р ИСО 14001 |
| `04_pressure_equipment` | Оборудование под давлением | Решение КТС №878 |
| `05_automation` | Автоматизация | ГОСТ Р 53684-2009, ПУЭ |
| `06_labor` | Охрана труда | ТК РФ, ФЗ-426, ГОСТ Р ИСО 45001 |
| `07_fire_safety` | Пожарная безопасность | ФЗ-69, ФЗ-123, СП 4.13130.2013 |
| `08_energy` | Энергетика | ФЗ-35, Приказ Минэнерго №811 |

### Автоматизация

**Скачивание документов (требует доступа к Consultant.ru):**
```bash
python scripts/download_normative_docs.py
```

**Аудит актуальности (ежеквартально):**
```bash
python scripts/audit_normative_docs.py
```

Отчёты об аудите сохраняются в `data/01_raw/normative_base/audit_report_YYYYMMDD.md`.

### Периодическое обслуживание

Настройте cron/systemd для автоматического аудита раз в квартал. Скрипт проверяет статус документов на Consultant.ru и генерирует отчёт о:
- утративших силу документах (требуют замены)
- доступных новых редакциях
- непроверенных документах
