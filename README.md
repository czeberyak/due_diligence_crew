# 🏭 Due Diligence Crew

**Multi-agent система на базе CrewAI и OpenRouter для автоматизированного Due Diligence инвестиционных ТЭО**

Проект реализует концепцию **экспертного консилиума** под управлением LLM-менеджера, который автоматизирует глубокий технический, экологический и финансовый аудит проектной документации крупных промышленных объектов.

---

## 🎯 Ключевые возможности

- **🔍 Глубина аудита:** Полный комплаенс-контроль проектной документации на соответствие ФНП Ростехнадзора, ГОСТ, ТУ и ISO.
- **⚖️ Физическая верификация:** Автоматический сквозной расчет материально-тепловых балансов с жестким отсевом невязок (порог > 5%) и детектированием нарушения закона сохранения массы (выход > 100%).
- **📜 Multi-Document RAG:** Автоматический кросс-чекинг ТЭО с реальной базой нормативных актов (Приказы, ГОСТ, ТУ) из локальной папки.
- **💰 Экономический перевод:** Маппинг инженерных уязвимостей в стоимостную оценку рисков модернизации, штрафов и изменения CAPEX/OPEX.
- **🌐 Архитектурная отказоустойчивость:** Полная независимость от прямых зарубежных блокировок OpenAI за счет интеграции шлюза OpenRouter.
- **📊 Иерархическое управление:** LLM-менеджер самостоятельно декомпозирует задачу, распределяет подзадачи между экспертами и проводит кросс-валидацию.

## 🏗️ Архитектура консилиума

Система построена по канонам модульных production-систем с разделением фабрик агентов, логики задач и расчетных инструментов:

```
due_diligence_crew/
├── data/
│   ├── 01_raw/              # Исходные PDF-файлы ТЭО + нормативные акты
│   ├── 02_processed/        # Сгенерированные отчеты (Markdown)
│   └── vector_db/           # Векторная база ChromaDB (после индексации)
├── src/
│   ├── agents.py            # Фабрика агентов (Auditor, Engineer, Analyst)
│   ├── tasks.py             # Конфигурация конвейера задач и иерархических связей
│   ├── tools.py             # Кастомные инструменты (PDF-парсер, Мат. валидатор, Reg. поиск)
│   ├── ingest.py            # Скрипт индексации PDF в векторную БД (ChromaDB)
│   └── main.py              # Точка входа, конфигурация LLM и запуск Crew
├── .env                     # Переменные окружения и API-ключи
├── check_imports.py         # Модуль предполётной валидации
└── requirements.txt         # Зависимости (CrewAI, Pydantic v2, pdfplumber, chromadb)
```

## 🛠️ Кастомные инструменты

### `DocumentSearchTool`
Честный парсер проектной документации на базе `pdfplumber`. Постранично извлекает текст и **технологические таблицы** (критично для материальных балансов!) из PDF, фильтруя данные по ключевым словам. Поддерживает защитные механизмы от системных файлов Windows и лимит на объем токенов (15 000 символов).

### `RegulatorySearchTool`
Инструмент Multi-Document RAG для сверки с законодательством. Сканирует **все PDF-файлы** в папке `data/01_raw/` (кроме самого ТЭО), ищет по ключевым словам в реальных текстах нормативных актов (Приказы Ростехнадзора, ГОСТ, ТУ, Решения Комиссий ТС). Позволяет агенту цитировать конкретные требования законов вместо галлюцинаций.

### `ProcessCalculatorTool`
Жестко детерминированный верификатор на чистом Python. Агент-технолог **обязан** прогонять числовые данные схем через этот инструмент. Автоматически вычисляет:
- Суммарные потоки сырья и продуктов
- Процент невязки материального баланса
- Фактический vs заявленный выход целевого продукта
- **Вердикт:** при невязке > 5% — автоматический флаг физической нереализуемости проекта
- **Защита от «создания материи»:** если масса продуктов > массы сырья — критическая ошибка физики

## 🏆 Результаты успешных прогонов (Case Study)

Система провела серию итеративных Due Diligence ТЭО проекта строительства завода по производству полиэтилена низкого давления (500 000 т/год). Ниже — эволюция качества аудита.

### Итерация 1: Базовый аудит (без Regulatory Search)
| Показатель | Заявлено в ТЭО | Выявлено консилиумом |
|------------|----------------|----------------------|
| **Выход целевого продукта** | 95% | **91.24%** (отклонение -3.76%) |
| **Невязка материального баланса** | — | **2.37%** (в пределах нормы ✅) |
| **Базовый CAPEX** | 95 млрд руб. | $1 200 млн (USD, галлюцинация валюты) |
| **IRR (с учётом рисков)** | 18.5% | **≈ 12%** |
| **Срок окупаемости** | 6.2 года | **9.5 года** |

> ⚠️ **Проблемы:** Агент не читал реальные законы, галлюцинировал банковские ГОСТы (ГОСТ Р 57580-2017, Положение Банка России № 777-П), конвертировал рубли в USD без обоснования.

### Итерация 2: Multi-Document RAG (с Regulatory Search)
| Показатель | Результат |
|------------|-----------|
| **Невязка баланса** | **5.96%** — превышает порог 5% → требуется корректировка |
| **Выявленные несоответствия** | **7 критических пунктов** (ПСУ ПБ, ПМ ОПБ, ПТОТР, ПОО, кибер-безопасность, ПДК выбросов, ПДК сточных вод) |
| **Доп. CAPEX** | **+445 млн ₽** на устранение несоответствий |
| **Доп. OPEX** | **+38 млн ₽/год** |
| **Рекомендация** | **Приостановить финансирование** до устранения критических несоответствий |

> ✅ **Прогресс:** Агент начал читать реальные PDF-файлы законов, исчезли галлюцинации, появились конкретные требования (ПСУ ПБ, ПМ ОПБ, ПТОТР).

### Итерация 3: Физическая верификация + Усиленные промпты
| Показатель | Результат |
|------------|-----------|
| **Выход продукта** | **104.2%** (масса продуктов > массы сырья) |
| **Вердикт process_calculator** | ❌ **КРИТИЧЕСКАЯ ОШИБКА ФИЗИКИ: проект нереализуем** |
| **Финансовый эффект** | **100% риск потери инвестиций** |
| **Выявленные документы** | ПСУ ПБ, ПМ ОПБ, ПТОТР, АОР/HAZOP, ПЧС, САО, ПМР, ПОО |
| **Рекомендация** | **Корректировка материального баланса обязательна** |

> 🏆 **Финальное качество:** Система научилась блокировать физически невозможные проекты, честно отказываться от выдуманных цифр («Критическое правило математики») и формировать матрицу рисков уровня Big4.

## 🚀 Быстрый старт

### 1. Клонирование и установка зависимостей

```bash
git clone https://github.com/czeberyak/due_diligence_crew.git
cd due_diligence_crew
python3 -m venv .venv
source .venv/bin/activate  # для Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Конфигурация окружения

Создайте файл `.env` в корне проекта:

```env
OPENROUTER_API_KEY=sk-or-v1-ваш-ключ-из-панели-openrouter
CREWAI_TRACING_ENABLED=false
```

### 3. Предполётная проверка

```bash
python check_imports.py
```

Ожидаемый результат: `✅ Все базовые компоненты архитектуры связаны корректно!`

### 4. (Опционально) Индексация документов в ChromaDB

```bash
python src/ingest.py
```

### 5. Запуск консилиума

Поместите ваш PDF-файл с ТЭО в `data/01_raw/` и выполните:

```bash
python src/main.py
```

## 🔧 Технологический стек

| Компонент | Технология | Назначение |
|-----------|------------|------------|
| Orchestration | **CrewAI** | Управление агентами и задачами (иерархический процесс) |
| LLM Gateway | **OpenRouter** | Доступ к GPT-4o, Llama 3.3, Gemini и др. |
| LLM Model | **gpt-oss-120b:free** | Бесплатная модель для всех агентов и менеджера |
| PDF Parsing | **pdfplumber** | Извлечение текста и таблиц из PDF |
| Vector DB | **ChromaDB** | Векторное хранилище для семантического поиска |
| Embeddings | **all-MiniLM-L6-v2** | Встроенная модель эмбеддингов ChromaDB |
| Data Validation | **Pydantic v2** | Строгая типизация инструментов |
| Environment | **python-dotenv** | Безопасное хранение секретов |
