# 🕵️‍♂️ Автоматизированный Иерархический Консилиум Due Diligence Инвестиционных ТЭО

Multi-agent система на базе **CrewAI** и **OpenRouter** для проведения глубокого технического, экологического и финансового аудита инвестиционных технико-экономических обоснований (ТЭО) крупных промышленных и химических объектов. Проект реализует концепцию экспертного консилиума под управлением LLM-менеджера, автоматизируя выявление технологических рисков, проверку законов сохранения массы и трансляцию инженерных дефектов в финансовые метрики (CAPEX/OPEX).

## 🎯 Техническое задание системы (Target Profile)

* **Глубина аудита:** Полный комплаенс-контроль проектной документации на соответствие ФНП (Федеральным нормам и правилам) Ростехнадзора, экологическим регламентам и ГОСТ/ТУ.
* **Физическая верификация:** Автоматический сквозной расчет материально-тепловых балансов химико-технологических схем с жестким отсевом невязок.
* **Экономический перевод:** Маппинг инженерных уязвимостей в стоимостную оценку рисков модернизации, штрафов и изменения операционных затрат.
* **Архитектурная отказоустойчивость:** Полная независимость от прямых зарубежных блокировок OpenAI за счет интеграции шлюзов OpenRouter.

## 🏗 Архитектурная структура проекта

Репозиторий спроектирован по канонам модульных production-систем с разделением фабрик агентов, логики задач и расчетных инструментов:

```text
due_diligence_crew/
├── data/
│   └── 01_raw/              # Исходные PDF-файлы ТЭО инвестиционных проектов
├── src/
│   ├── agents.py            # Фабрика декларации агентов (Auditor, Engineer, Analyst)
│   ├── tasks.py             # Конфигурация конвейера задач и иерархических связей
│   ├── tools.py             # Кастомные инструменты (Математический валидатор баланса, RAG)
│   └── main.py              # Точка входа, конфигурация LLM-провайдера и запуск Crew
├── .env                     # Переменные окружения и секретные ключи API
├── .gitignore               # Исключения системного мусора, локального кэша и секретов
├── check_imports.py         # Модуль предполётной валидации связности архитектуры
└── requirements.txt         # Декларация зависимостей (CrewAI, Pydantic v2, Dotenv)

```

## 🚀 Ключевые инженерные решения

### 1. Иерархическая AI-оркестрация (Hierarchical Crew Process)

В отличие от стандартных линейных цепочек (Sequential), управление проектом делегировано выделенному **LLM-Менеджеру** (на базе передовых моделей). Менеджер:

* Самостоятельно декомпозирует глобальную задачу Due Diligence.
* Распределяет подзадачи между узкоспециализированными агентами в зависимости от их бэкграунда.
* Проводит кросс-валидацию ответов, возвращая отчеты на доработку в случае обнаружения логических пробелов.

### 2. Physics-Informed Валидация (`ProcessCalculatorTool`)

Для защиты от классических галлюцинаций больших языковых моделей разработан жестко детерминированный инструмент верификации на чистом Python (`process_calculator`). Агент-технолог обязан прогонять числовые данные схем через этот инструмент. Он автоматически вычисляет:

* Суммарные потоки сырья и продуктов.
* Процент невязки материального баланса согласно закону сохранения массы.
* **Критический порог:** Если ошибка невязки превышает **5.0%**, система автоматически выставляет проекту вердикт физической нереализуемости или подлога в цифрах.

### 3. Стабилизированный интерфейс OpenRouter через нативный `CrewAI.LLM`

Перенос вызовов на интерфейс OpenRouter реализован через официальный класс `LLM` экосистемы CrewAI, что решило проблему совместимости со спецификацией OpenAI Function Calling:

* Обеспечен строгий ASCII-комплаенс имен функций (исключена кириллица и пробелы в Pydantic-схемах инструментов).
* Реализована гибкая конфигурация: тяжелые интеллектуальные модели (например, `openai/gpt-4o`) жестко пинятся на координацию менеджера, а сверхдешевые и быстрые (`openai/gpt-4o-mini`, `gemini-2.0-flash`) автоматизируют работу линейных экспертов.

## 📊 Текущий статус и Roadmap

*  Этап 1: Скелет и Проводка. Разработана модульная структура проекта, успешно сопряжены фабрики агентов и задач.
*  Этап 2: Валидация целостности. Написан скрипт `check_imports.py` для тестирования импортов и контроля наличия API-ключей в `.env`.
*  Этап 3: Подключение ядра. Интегрирован шлюз OpenRouter, побеждена ошибка пустых имен инструментов при сериализации схем в JSON.
*  Этап 4: Симуляционный прогон. Успешно выполнен первый тестовый запуск консилиума на пустом файле-заглушке, подтвердивший работоспособность иерархического процесса.
*  Этап 5: Реализация RAG-пайплайна. Замена заглушки `document_search` в `tools.py` на реальный парсер текстовых и табличных слоев PDF (`pdfplumber` / `pypdf`) и векторизацию контекста.
*  Этап 6: Streamlit WebUI. Разработка локального интерактивного дашборда для удобной загрузки ТЭО и вывода логов работы агентов в реальном времени.
- **Этап 7: End-to-End прогон мультиагентного консилиума (MVP Ready ✅).** 
Успешно протестирована полная цепочка Due Diligence на реальном многостраничном PDF (ТЭО завода полиэтилена). 
- `Safety Auditor` успешно парсит документ, выявляет риски и привязывает их к регуляторным актам (Ростехнадзор, ГОСТ, ISO).
- `Process Engineer` извлекает сырьевые потоки из таблиц, передает их в `process_calculator` и верифицирует закон сохранения массы (невязка баланса).
- `Investment Analyst` агрегирует данные коллег и переводит инженерные уязвимости в финансовую плоскость (корректировка CAPEX/OPEX, расчет IRR).

## ⚙️ Установка и запуск

### Клонирование и настройка окружения

```bash
git clone <url_вашего_репозитория> && cd due_diligence_crew
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

```

### Настройка секретов (`.env`)

Создайте файл `.env` в корневой директории проекта и добавьте ваш токен:

```text
OPENROUTER_API_KEY=sk-or-v1-ваш-ключ-из-панели-openrouter

```

### Верификация и запуск системы

1. Запустите экспресс-тест предполётной проверки целостности:
```bash
python check_imports.py

```


2. Если проверка завершилась статусом `✅ Все базовые компоненты архитектуры связаны корректно!`, запускайте боевой цикл консилиума:
```bash
python src/main.py

```
